Windows Migration

Migrate Hermes Agent data from C: drive to any custom directory — zero deletion, incremental copy, auto env-var setup.

🏷️ v1.0.1 🔧 Hermes Agent 🪟 Windows Only
⚡ 快速安装
# 安装技能
hermes skills tap add jove-rina/jove-hermes-skills

# 安装迁移技能
hermes skills install jove-rina/jove-hermes-skills/hermes-windows-migration

# 加载并执行
/skill hermes-windows-migration
特性

核心特性

六大特性让 Windows 迁移安全、高效、零风险

🗂️

Zero Deletion

Copies only, never deletes original data. Safe to try — rollback is instant.

📦

Incremental Copy

Uses robocopy with multi-thread, resume support, ACL & file attributes preserved.

🎯

Auto-Detection

Smartly locates source by checking existing HERMES_HOME env var. No manual path guessing.

⚙️

Auto Env-Var Setup

Sets user-level HERMES_HOME environment variable. No admin or PowerShell expertise needed.

✅

Verification Checklist

Post-migration output shows step-by-step verification: paths, env var, data integrity.

🪟

Windows Only

Designed for Windows native install. Not for WSL2, Linux, or macOS.

方案

问题 vs 解决方案

C: 盘空间不足？手动迁移风险高。这个技能自动化了整个流程。

✕ 问题

C: 盘空间逐渐被占满

Hermes 默认将数据存储在 C:\Users\<user>\AppData\Local\hermes。随着使用时间推移，日志、缓存和配置数据可能占用数 GB 空间，逐渐填满 C: 盘。手动迁移需要修改环境变量、复制文件、确保路径正确——任何一步出错都可能导致数据丢失或 Hermes 无法启动，且需要了解 PowerShell 管理权限操作。

✓ 解决方案

全自动迁移 — 零数据丢失

此 Hermes Agent 技能提供全自动迁移方案：使用 robocopy 进行快速增量传输，零数据丢失；自动检测现有 HERMES_HOME 设置；自动配置用户级 HERMES_HOME 环境变量。提供三种方式可选：一键自动化脚本、手动 PowerShell 步骤、或 AI 全引导迁移。无需管理员权限或 PowerShell 专业知识。
